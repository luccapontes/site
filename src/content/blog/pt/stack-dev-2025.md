## Por que escrever sobre stack?

Não é pra mostrar que uso a tecnologia certa. É pra documentar o raciocínio por trás das escolhas — que é o que realmente importa.

## Frontend

**Astro** — para sites de conteúdo como este. Zero JavaScript por padrão, ótimo para SEO, e a experiência de desenvolvimento é muito boa.

**React** — quando preciso de interatividade real. Ainda é o padrão na maioria dos projetos de clientes.

## Backend

**Node.js + TypeScript** — minha combinação de conforto. TypeScript me salvou de bugs em produção inúmeras vezes.

**PostgreSQL** — banco relacional que escala bem e tem features poderosas (JSONb, full-text search) que evitam dependências extras.

## Infraestrutura

**Vercel** — para projetos front-heavy. Deploy em segundos, domínios simples, edge functions quando preciso.

**Railway** — para backends com banco de dados. Mais simples que AWS para a maioria dos casos de clientes.

## O que aprendi

Stack não é identidade. Use o que resolve o problema. Mas conhecer bem uma stack específica te torna mais rápido e seguro do que usar muitas coisas superficialmente.
